Línea de Comandos Parte 2

10.1 - Visualización de Contenido

Visualizar y examinar archivos

Una vez que sabes manipular archivos, el siguiente paso es poder ver y examinar su contenido de diferentes maneras.

En esta sección aprenderás:

  • Mostrar todo el contenido de archivos
  • Leer archivos largos página por página
  • Ver solo las primeras o últimas líneas
  • Obtener estadísticas y detalles de archivos
cat less/more head/tail wc/file/stat

cat - Mostrar contenido completo

El comando cat (concatenate) muestra todo el contenido de uno o más archivos directamente en la terminal.

Uso básico de cat

usuario@mint:~$ cat archivo.txt
Contenido completo del archivo.txt
Se muestra línea por línea
Hasta el final del archivo

usuario@mint:~$ cat archivo1.txt archivo2.txt
# Muestra el contenido de ambos archivos concatenados

usuario@mint:~$ cat *.txt
# Muestra todos los archivos .txt en orden

Función: Muestra el contenido completo de archivos en la salida estándar.
Ideal para: Archivos pequeños, scripts, archivos de configuración.
Precaución: No uses cat con archivos muy largos, saturará la terminal.

Opciones útiles de cat

usuario@mint:~$ cat -n archivo.txt
1 Primera línea numerada
2 Segunda línea numerada
3 Tercera línea numerada

usuario@mint:~$ cat -A archivo.txt
# Muestra caracteres no imprimibles (espacios, tabs, saltos de línea)

usuario@mint:~$ cat -s archivo.txt
# Suprime líneas en blanco múltiples consecutivas

Opciones principales:
- -n: Numera todas las líneas
- -b: Numera solo las líneas que no están en blanco
- -A: Muestra caracteres especiales (tabs como ^I, fin de línea como $)
- -s: Suprime líneas en blanco múltiples
- -T: Muestra tabs como ^I

less y more - Lectura por páginas

Para archivos largos, less y more permiten navegar por el contenido página por página.

less - El paginador avanzado (recomendado)

usuario@mint:~$ less archivo_largo.txt
# Abre el archivo en modo interactivo
── Contenido del archivo mostrado página por página ──
Usa las teclas para navegar:
:
Controles de navegación en less:
Tecla Acción
Espacio o f Página siguiente
b Página anterior
↓ o j Línea siguiente
↑ o k Línea anterior
g Ir al inicio del archivo
G Ir al final del archivo
/texto Buscar "texto" hacia adelante
?texto Buscar "texto" hacia atrás
n Siguiente coincidencia de búsqueda
q Salir de less

Ventajas de less:
- Permite navegar hacia adelante y hacia atrás
- Función de búsqueda integrada
- No carga todo el archivo en memoria (eficiente con archivos grandes)
- Resaltado de sintaxis básico

more - El paginador básico

usuario@mint:~$ more archivo.txt
# Muestra el archivo página por página (solo hacia adelante)
-- More -- (25%)
# Presiona Espacio para continuar, q para salir

Características de more:
- Solo permite avanzar (no retroceder)
- Más simple que less
- Muestra porcentaje de progreso
- Controles: Espacio (siguiente página), Enter (siguiente línea), q (salir)

Recomendación: Usa less en lugar de more porque es más potente y flexible. La mayoría de sistemas Linux modernos tienen less instalado por defecto.

head y tail - Primeras o últimas líneas

Cuando solo necesitas ver el inicio o final de un archivo, head y tail son perfectos.

head - Primeras líneas

usuario@mint:~$ head archivo.txt
# Muestra las primeras 10 líneas (por defecto)

usuario@mint:~$ head -n 5 archivo.txt
# Muestra solo las primeras 5 líneas

usuario@mint:~$ head -20 archivo.txt
# Muestra las primeras 20 líneas

Opciones de head:
- -n NUM: Muestra NUM líneas
- -c NUM: Muestra NUM caracteres
- -q: No muestra nombres de archivo
- -v: Siempre muestra nombres de archivo

tail - Últimas líneas

usuario@mint:~$ tail archivo.txt
# Muestra las últimas 10 líneas (por defecto)

usuario@mint:~$ tail -n 15 /var/log/syslog
# Muestra las últimas 15 líneas del log del sistema

usuario@mint:~$ tail -f /var/log/syslog
# Sigue el archivo en tiempo real (útil para logs)

Opciones de tail:
- -n NUM: Muestra NUM líneas
- -c NUM: Muestra NUM caracteres
- -f: Sigue el archivo (follow)
- -F: Como -f pero maneja rotación de logs

Ejemplos prácticos con head y tail

usuario@mint:~$ head -1 *.txt
# Muestra la primera línea de todos los archivos .txt

usuario@mint:~$ tail -5 archivo1.txt archivo2.txt
# Muestra las últimas 5 líneas de ambos archivos

usuario@mint:~$ head -50 archivo.txt | tail -10
# Muestra las líneas 41-50 (combina head y tail con pipe)

usuario@mint:~$ tail -f /var/log/auth.log
# Monitorea intentos de login en tiempo real (Ctrl+C para salir)

wc, file y stat - Estadísticas y detalles

Estos comandos te dan información detallada sobre archivos sin mostrar su contenido.

wc - Contar palabras, líneas y caracteres

usuario@mint:~$ wc archivo.txt
42 156 1024 archivo.txt
# líneas palabras caracteres nombre

usuario@mint:~$ wc -l *.txt
# Cuenta solo las líneas de todos los archivos .txt

usuario@mint:~$ wc -w ensayo.txt
1247 ensayo.txt
# Cuenta solo las palabras

Opciones de wc:
- -l: Solo cuenta líneas
- -w: Solo cuenta palabras
- -c: Solo cuenta caracteres (bytes)
- -m: Solo cuenta caracteres (incluyendo multibyte)
- -L: Longitud de la línea más larga

file - Identificar tipo de archivo

usuario@mint:~$ file documento.txt
documento.txt: UTF-8 Unicode text

usuario@mint:~$ file imagen.jpg
imagen.jpg: JPEG image data, JFIF standard 1.01

usuario@mint:~$ file script.sh
script.sh: Bourne-Again shell script, ASCII text executable

usuario@mint:~$ file *
# Identifica el tipo de todos los archivos en el directorio actual

Función: Determina el tipo de archivo analizando su contenido, no solo la extensión.
Útil para: Identificar archivos sin extensión, verificar si un archivo es realmente lo que parece ser, detectar archivos corruptos o binarios que parecen texto.

stat - Información detallada del archivo

usuario@mint:~$ stat archivo.txt
File: archivo.txt
Size: 1024 Blocks: 8 IO Block: 4096 regular file
Device: 801h/2049d Inode: 123456 Links: 1
Access: (0644/-rw-r--r--) Uid: (1000/usuario) Gid: (1000/usuario)
Access: 2025-01-15 10:30:25.123456789 -0300
Modify: 2025-01-15 10:25:30.987654321 -0300
Change: 2025-01-15 10:25:30.987654321 -0300
Birth: -

usuario@mint:~$ stat -c "%n: %s bytes, %y" *.txt
# Formato personalizado: nombre, tamaño y fecha de modificación

Información que proporciona stat:
- Nombre y tamaño del archivo
- Tipo de archivo y permisos
- Propietario y grupo
- Fechas de acceso, modificación y cambio de metadatos
- Número de inode y enlaces
- Información del dispositivo donde está almacenado

Práctica guiada

Ejercicios básicos para practicar visualización de archivos:

Paso inicial: Abre una terminal y ve a tu carpeta personal con cd ~
Ejercicio 1: Crear y ver un archivo simple
  1. Crea un archivo de prueba: echo "Hola mundo" > mi_archivo.txt
  2. Ve el contenido con: cat mi_archivo.txt
  3. Agrega más contenido: echo "Segunda línea" >> mi_archivo.txt
  4. Ve el archivo completo de nuevo: cat mi_archivo.txt
Ejercicio 2: Practicar con head y tail
  1. Crea un archivo con más líneas:
    echo "Línea 1" > archivo_largo.txt
    echo "Línea 2" >> archivo_largo.txt
    echo "Línea 3" >> archivo_largo.txt
    echo "Línea 4" >> archivo_largo.txt
    echo "Línea 5" >> archivo_largo.txt
  2. Ve las primeras 2 líneas: head -2 archivo_largo.txt
  3. Ve las últimas 2 líneas: tail -2 archivo_largo.txt
  4. Ve todo el archivo numerado: cat -n archivo_largo.txt
Ejercicio 3: Contar y examinar archivos
  1. Cuenta las líneas de tu archivo: wc -l archivo_largo.txt
  2. Cuenta las palabras: wc -w archivo_largo.txt
  3. Ve qué tipo de archivo es: file archivo_largo.txt
  4. Ve la información básica: ls -l archivo_largo.txt
Ejercicio 4: Explorar archivos del sistema (fácil)
  1. Ve las primeras líneas de tu perfil de bash: head -5 ~/.bashrc
  2. Ve información de tu sistema: head -3 /etc/os-release
  3. Cuenta cuántas líneas tiene: wc -l /etc/os-release
¡Bien hecho! Una vez que domines estos ejercicios básicos, estarás listo para explorar archivos más complejos y usar comandos como less para archivos largos.
Recuerda: Si te pierdes o algo no funciona, siempre puedes usar ls para ver qué archivos tienes en tu directorio actual.

Resumen comparativo

Comando Función Mejor uso Precaución
cat Muestra todo el contenido Archivos pequeños, scripts No usar con archivos muy grandes
less Paginador interactivo Archivos largos, navegación Recordar salir con 'q'
more Paginador básico Sistemas sin less Solo avanza, no retrocede
head Primeras líneas Ver inicio de logs, archivos Por defecto solo 10 líneas
tail Últimas líneas Logs recientes, monitoreo -f consume recursos
wc Contar líneas/palabras Estadísticas de texto Los espacios afectan el conteo
file Identificar tipo Archivos sin extensión Puede no ser 100% preciso
stat Metadatos completos Información detallada Mucha información
